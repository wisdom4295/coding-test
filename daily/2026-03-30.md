## 복기. Valid Anagram — Easy · Hash Map

🔗 https://leetcode.com/problems/valid-anagram/

### 내가 작성한 코드 (오답)

```js
function getMapCount(s){
  const tempObj = {};
  
  for(let i=0; i < s.length; i++){
    if(s[i] in tempObj){
      tempObj[s[i]] += 1;   // ✅ 키 접근 올바름
    }
    tempObj[s[i]] = 1;      // ❌ else 없이 항상 1로 덮어씀
  }
  
  return tempObj            // ✅ 루프 바깥 return (진전!)
}

function isAnagram(s, t) {   
  const objS = getMapCount(s).keys().sort();  // ❌ 일반 객체에 .keys() 없음
  const objT = getMapCount(t).keys().sort();  // ❌ keys()가 있어도 sort() 반환값 문제
  
  if(JSON.stringify(objS) === JSON.stringify(objT)) return true;
  return false;
};
```

---

### 틀린 이유

**✅ 진전된 점 먼저**

```
저번까지: tempObj += 1  (객체에 직접 연산 → 타입 오염)
오늘:     tempObj[s[i]] += 1  (키를 통해 올바르게 접근!)
return도 루프 바깥으로 이동 → 4일 만에 해결 ✅
```

**❌ 버그 1: `else` 없이 항상 `tempObj[s[i]] = 1`로 덮어씀**

```js
if(s[i] in tempObj){
  tempObj[s[i]] += 1;  // 증가시켜도
}
tempObj[s[i]] = 1;     // ← else 없이 바로 1로 초기화해버림
                        // → 모든 문자 빈도수가 항상 1

// ✅ 올바른 구조: else로 분기
if(s[i] in tempObj){
  tempObj[s[i]] += 1;
} else {
  tempObj[s[i]] = 1;   // else 안에만 실행되어야 함
}
```

**❌ 버그 2: 일반 객체 `{}`에는 `.keys()` 메서드가 없음**

```js
const tempObj = {};       // 일반 객체
tempObj.keys()            // ❌ TypeError: tempObj.keys is not a function

// .keys()는 Map에만 있는 메서드
const map = new Map();
map.keys()                // ✅ 가능

// 일반 객체의 키를 배열로 얻으려면:
Object.keys(tempObj)      // ✅ ["a", "n", "g", ...]
```

**❌ 버그 3: `.keys()` 결과를 `.sort()`해도 빈도수 비교가 안 됨**

```js
// 키만 정렬해서 비교하면 "키가 같은가"만 확인 → 빈도수(값) 비교 안 됨
// 예시: s="aab", t="abb"
// Object.keys 결과: 둘 다 ["a", "b"] → 정렬해도 같음
// 그러나 빈도수는 다름: { a:2, b:1 } vs { a:1, b:2 }
// → 같다고 잘못 판단

// ✅ 객체 전체(키+값)를 비교해야 함
JSON.stringify(getMapCount(s)) === JSON.stringify(getMapCount(t))
```

---

### 🧩 핵심 접근법 — Hash Map (O(n))

**접근 순서**

```
Step 1. 길이가 다르면 → 즉시 false
Step 2. 각 문자열의 빈도수 객체 생성
Step 3. JSON.stringify로 객체 전체 비교 (키 + 값 모두)
```

---

### 올바른 풀이

```js
function getMapCount(s) {
  const tempObj = {};
  
  for (let i = 0; i < s.length; i++) {
    if (s[i] in tempObj) {
      tempObj[s[i]] += 1;
    } else {              // ✅ else로 분기
      tempObj[s[i]] = 1;
    }
  }
  
  return tempObj;         // ✅ 루프 바깥
}

function isAnagram(s, t) {
  if (s.length !== t.length) return false;
  // ✅ 객체 전체(키+값) 비교
  return JSON.stringify(getMapCount(s)) === JSON.stringify(getMapCount(t));
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 각 문자열 1회 순회 |
| 공간 | O(1) — 알파벳 최대 26개 키 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- `tempObj[s[i]] += 1` — 키를 통한 접근 드디어 정확히 작성! (4일 만에 해결 🎉)
- `return tempObj`를 루프 바깥으로 이동 — 이것도 오늘 완벽히 해결! 🎉
- `JSON.stringify`로 비교하려는 방향 맞음

**보완할 점**
- `if` 다음에 반드시 `else` 붙이기: `else` 없으면 조건과 무관하게 항상 실행됨
- 일반 객체 `{}`와 `Map`의 차이:
  ```js
  {}       → Object.keys(obj), obj[key]
  new Map() → map.keys(), map.get(key), map.set(key, val)
  ```
- 키만 비교하면 값(빈도수)이 다를 때를 잡지 못함 → `JSON.stringify` 전체 비교

## 1번. Product of Array Except Self — Medium · Array

🔗 https://leetcode.com/problems/product-of-array-except-self/

### 문제 요구사항

- 정수 배열 `nums`가 주어졌을 때
- `result[i]` = `nums[i]`를 **제외한** 나머지 모든 요소의 곱
- **나눗셈을 사용하지 않고** O(n)으로 풀기

---

### 내가 작성한 코드 (오답)

```ts
function productExceptSelf(nums: number[]): number[] {
  const tempArray = [];
  let totalMultiple = 1;
  
  for(let i = 0; i < nums.length; i++){
    totalMultiple *= nums[i];   // 전체 곱 계산
  }
  
  for(let i = 0; i < nums.length; i++){
     tempArray.push(totalMultiple / nums[i]);  // ❌ 나눗셈 사용
  }
    return tempArray;
};
```

---

### 틀린 이유

**❌ 버그 1: 나눗셈 사용 — 문제 조건 위반**

```
문제 조건: "나눗셈을 사용하지 말 것"
→ totalMultiple / nums[i] 자체가 제약 조건 위반
```

**❌ 버그 2: `nums[i] === 0`이 있을 때 나눗셈 불가**

```js
nums = [1, 0, 3, 4]
totalMultiple = 0

totalMultiple / nums[1] = 0 / 0 = NaN  // 0으로 나누기
→ 결과가 NaN이 됨
```

**❌ 버그 3: 0이 2개 이상이면 모든 결과가 0**

```js
nums = [0, 0, 3, 4]
→ 어느 인덱스를 제외해도 나머지 곱에 0이 포함
→ 전체 곱을 나눗셈으로 구하면 처리 불가
```

---

### 🧩 핵심 접근법 — 좌우 누적곱 (O(n))

**왜 좌우 누적곱인가?**

나눗셈 없이 `nums[i]`를 제외한 곱을 구하려면:
- `nums[i]` **왼쪽** 요소들의 곱 × `nums[i]` **오른쪽** 요소들의 곱

왼쪽 → 오른쪽, 오른쪽 → 왼쪽 두 번 순회하면 O(n)에 해결 가능.

**접근 순서**

```
Step 1. result[i] = nums[0] * ... * nums[i-1]  (왼쪽 누적곱)
         result[0] = 1 (왼쪽에 아무것도 없음)

Step 2. 오른쪽에서 누적곱 R을 유지하면서 result[i] *= R
         R = nums[i+1] * ... * nums[n-1]
         result[n-1]에서 시작, R = 1 (오른쪽에 아무것도 없음)
```

**핵심 아이디어 시각화**

```
nums    = [1, 2, 3, 4]

Step 1. 왼쪽 누적곱:
result  = [1, 1, 2, 6]
          ↑  ↑  ↑  ↑
          1  1  1×2 1×2×3

Step 2. 오른쪽 누적곱 R을 곱하며 갱신 (오른쪽 → 왼쪽):
i=3: result[3] *= 1(R)  → 6×1  = 6,  R = 4
i=2: result[2] *= 4(R)  → 2×4  = 8,  R = 4×3 = 12
i=1: result[1] *= 12(R) → 1×12 = 12, R = 12×2 = 24
i=0: result[0] *= 24(R) → 1×24 = 24

result = [24, 12, 8, 6] ✅
```

---

### 올바른 풀이

```ts
function productExceptSelf(nums: number[]): number[] {
  const n = nums.length;
  const result = new Array(n).fill(1);

  // Step 1. 왼쪽 누적곱
  let left = 1;
  for (let i = 0; i < n; i++) {
    result[i] = left;
    left *= nums[i];
  }

  // Step 2. 오른쪽 누적곱을 곱하며 갱신
  let right = 1;
  for (let i = n - 1; i >= 0; i--) {
    result[i] *= right;
    right *= nums[i];
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 2회 순회 |
| 공간 | O(1) — result 배열 제외 추가 공간 없음 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "전체 곱을 구하고 나눈다"는 직관적인 접근 → 나눗셈이 막혔을 때 다음 단계로 가는 발상 연습 필요
- 2중 for 루프 구조 자체는 올바르게 설계됨

**보완할 점**
- 나눗셈 금지 제약 → **좌우 누적곱** 패턴으로 전환하는 발상 익히기
- 핵심 공식:
  ```js
  // 왼쪽 누적곱 먼저 채우고
  result[i] = left; left *= nums[i];
  // 오른쪽에서 역방향으로 곱하며 갱신
  result[i] *= right; right *= nums[i];
  ```
- 0이 포함된 배열에서 나눗셈이 왜 안 되는지 인식: `0 / 0 = NaN`

---

### 틀린 이유

**❌ 버그 1: 정렬 없이 순서대로 비교 → 겹침 감지 불가**

```
intervals = [[1,3],[8,10],[2,6]]

정렬 없이 순서대로 비교하면:
[1,3]과 [8,10] 비교 → 안 겹침 → 따로 저장
[8,10]과 [2,6] 비교 → [2,6]이 앞인데 뒤에 나옴 → 감지 불가

반드시 시작값(start) 기준으로 정렬 후 비교해야 함
```

**❌ 버그 2: 핵심 로직 미구현**

```
"겹치면 합치고, 안 겹치면 새 구간 추가"하는 코드가 없음
→ 루프만 있고 아무것도 하지 않음
```

**❌ 버그 3: 겹침 조건 미정의**

```
두 구간 [a,b]와 [c,d]가 겹치는 조건:
c <= b  (다음 구간의 시작이 현재 구간의 끝보다 작거나 같으면 겹침)

겹치면: end = Math.max(b, d)  (더 큰 끝값으로 합침)
안 겹치면: 현재 구간을 결과에 push하고 다음 구간으로 교체
```

---

### 🧩 핵심 접근법 — 정렬 후 병합 (O(n log n))

**접근 순서**

```
Step 1. start 기준으로 정렬
Step 2. 첫 번째 구간을 결과 배열에 추가
Step 3. 나머지 구간을 순회하며:
    - 현재 구간의 start <= 결과 마지막 구간의 end → 겹침
      → 결과 마지막 구간의 end = Math.max(end, cur[1])
    - 안 겹치면 → 결과에 현재 구간 push
Step 4. 결과 반환
```

**핵심 아이디어 시각화**

```
intervals = [[1,3],[2,6],[8,10],[15,18]]

Step 1. 정렬 (이미 정렬됨)

Step 2. result = [[1,3]]

Step 3. 순회:
  [2,6]: 2 <= 3 → 겹침 → end = Math.max(3,6) = 6 → result = [[1,6]]
  [8,10]: 8 > 6 → 안 겹침 → push → result = [[1,6],[8,10]]
  [15,18]: 15 > 10 → 안 겹침 → push → result = [[1,6],[8,10],[15,18]]

결과: [[1,6],[8,10],[15,18]] ✅
```

---

### 올바른 풀이

```ts
function merge(intervals: number[][]): number[][] {
  // Step 1. start 기준 정렬
  intervals.sort((a, b) => a[0] - b[0]);

  const result: number[][] = [intervals[0]];

  for (let i = 1; i < intervals.length; i++) {
    const cur = intervals[i];
    const last = result[result.length - 1];

    if (cur[0] <= last[1]) {
      // 겹침 → end를 더 큰 값으로 갱신
      last[1] = Math.max(last[1], cur[1]);
    } else {
      // 안 겹침 → 새 구간 추가
      result.push(cur);
    }
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n log n) — 정렬 비용 |
| 공간 | O(n) — 결과 배열 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "작은 게 먼저, 큰 게 나중" → 정렬이 필요하다는 방향성 맞음
- 구간 비교가 필요하다는 것을 인식한 점

**보완할 점**
- 정렬을 코드로 바로 표현하기: `intervals.sort((a, b) => a[0] - b[0])`
- 겹침 조건 공식화: `cur[0] <= last[1]` → 겹침
- 겹칠 때 end 갱신: `last[1] = Math.max(last[1], cur[1])`
- 결과 배열에 첫 구간을 미리 넣고 시작하는 패턴 익히기
