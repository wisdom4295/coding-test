## 복기. Valid Anagram — Easy · Hash Map

🔗 https://leetcode.com/problems/valid-anagram/

### 내가 작성한 코드 (오답)

```js
function getMapCount(text){
  const tempObj = {};
  for(let i = 0; i < text.length; i++){
    const char = text[i]
    
    if(char in tempObj){
      tempObj += 1;    // ❌ 객체에 += 연산
    } else {
      tempObj = 1;     // ❌ 객체 자체에 값 할당
    }
    
    return tempObj;    // ❌ 루프 안에서 return
  }
}

function isAnagram(s, t) {
    return              // ❌ 반환값 없음
};
```

---

### 틀린 이유

**❌ 버그 1: `tempObj += 1` — 객체에 `+=` 연산 불가**

```js
// tempObj는 객체 {}
// tempObj += 1 → "[object Object]1" (문자열 변환)
// 의도한 건: tempObj[char] += 1

// ✅ 올바른 코드
tempObj[char] += 1;
```

**❌ 버그 2: `tempObj = 1` — 객체 자체를 덮어씀**

```js
// tempObj = 1 → 객체가 숫자 1로 교체됨
// 이후 모든 코드가 객체가 아닌 숫자 1에 대해 실행됨
// 의도한 건: tempObj[char] = 1

// ✅ 올바른 코드
tempObj[char] = 1;
```

**❌ 버그 3: `return tempObj`가 루프 안에 있음 (어제와 동일)**

```js
for(...) {
  ...
  return tempObj; // 첫 번째 반복에서 즉시 종료
                  // → 첫 글자만 카운트 후 반환
}

// ✅ 올바른 위치: 루프 바깥
for(...) { ... }
return tempObj;
```

**❌ 버그 4: `isAnagram`의 `return`에 반환값 없음**

```js
function isAnagram(s, t) {
    return  // ← undefined 반환
}

// ✅ 비교 결과를 return해야 함
return JSON.stringify(getMapCount(s)) === JSON.stringify(getMapCount(t));
```

---

### 🧩 핵심 접근법 — Hash Map (O(n))

**접근 순서**

```
Step 1. 길이가 다르면 → 즉시 false 반환
Step 2. 각 문자열의 빈도수를 객체로 카운트
Step 3. JSON.stringify로 객체 내용 비교 → 같으면 true
```

---

### 올바른 풀이

```js
function getMapCount(text) {
  const tempObj = {};
  
  for (let i = 0; i < text.length; i++) {
    const char = text[i];
    
    if (char in tempObj) {
      tempObj[char] += 1;  // ✅ 키에 접근해서 증가
    } else {
      tempObj[char] = 1;   // ✅ 키에 접근해서 초기화
    }
  }
  
  return tempObj; // ✅ 루프 완료 후 반환
}

function isAnagram(s, t) {
  if (s.length !== t.length) return false;
  return JSON.stringify(getMapCount(s)) === JSON.stringify(getMapCount(t)); // ✅ 내용 비교
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 각 문자열 1회 순회 |
| 공간 | O(1) — 알파벳 최대 26개 키 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- 어제보다 함수 구조와 for 루프 틀은 유지됨
- `char in tempObj` 조건 분기 방향 맞음

**보완할 점**
- `tempObj[char]`처럼 **반드시 키를 지정**해서 접근: 객체에 직접 `+=`, `=` 연산 불가
- `return`은 **루프 바깥**에: 이틀 연속 같은 실수 → 오늘 완전히 굳히기
- `return` 뒤에 반환할 **표현식** 붙이기: 빈 `return`은 `undefined`

## 1번. Top K Frequent Elements — Medium · Hash Map + Bucket Sort

🔗 https://leetcode.com/problems/top-k-frequent-elements/

### 문제 요구사항

- 정수 배열 `nums`와 정수 `k`가 주어졌을 때
- **가장 자주 등장하는 k개의 숫자** 반환 (순서 무관)

---

### 내가 작성한 코드 (미완성)

```ts
// k개의 길이만큼 빈 returnArray를 생성
// nums 배열을 돌면서 제일 많은 값을 차지 nums 배열의 요소를
// returnArray에 push 후 return

function getCountObj(numsArray){
  const count = {};

  for (const num of numsArray) {
    count[num] = (count[num] || 0) + 1;
  }

  return count;
}

function topKFrequent(nums: number[], k: number): number[] {
  const returnArr = [];
  returnArr.length = k;     // ❌ 빈 슬롯 k개 생성 (의미 없음)
  
  for(let i = 0; i < nums.length; i++){
    // ❌ 빈 루프: 아무 로직도 없음
  }
  
  return returnArr;         // ❌ 빈 배열 반환
};
```

---

### 틀린 이유

**❌ 버그 1: `returnArr.length = k` — 의미 없는 초기화**

```js
const arr = [];
arr.length = 3;
// → [empty × 3] (빈 슬롯만 생성, 값 없음)
// k개를 미리 잡아두려 했지만, push로 채울 거라면 이 코드는 오히려 방해됨
```

**❌ 버그 2: for 루프 안에 로직 없음**

```js
for(let i = 0; i < nums.length; i++){
  // 아무것도 없음 → returnArr는 영원히 비어 있음
}
```

**❌ 버그 3: 빈도수를 구했지만 "상위 k개 선별" 로직이 없음**

```
getCountObj()로 빈도수 객체는 만들었지만
→ 어떤 숫자가 몇 번 나왔는지 정렬하거나 선별하는 코드가 없음
→ 핵심인 "k개 뽑기"가 미구현
```

---

### 🧩 핵심 접근법 — Bucket Sort (O(n))

**왜 Bucket Sort인가?**

빈도수를 기준으로 정렬하면 O(n log n)이지만,
빈도수의 최댓값은 `nums.length`를 초과할 수 없다는 사실을 이용하면
인덱스 = 빈도수인 버킷 배열로 O(n)에 해결 가능하다.

**접근 순서**

```
Step 1. 빈도수 Map 생성: { 숫자: 등장 횟수 }
Step 2. 버킷 배열 생성: bucket[i] = 빈도수가 i인 숫자들의 배열
         (크기: nums.length + 1, 인덱스 = 빈도수)
Step 3. 버킷 배열을 역방향(높은 빈도 → 낮은 빈도)으로 순회
         → k개가 모일 때까지 결과 배열에 추가
Step 4. 결과 반환
```

**핵심 아이디어 시각화**

```
nums = [1,1,1,2,2,3], k = 2

Step 1. 빈도수 Map: { 1:3, 2:2, 3:1 }

Step 2. 버킷 배열 (인덱스 = 빈도수):
  bucket[0] = []
  bucket[1] = [3]
  bucket[2] = [2]
  bucket[3] = [1]

Step 3. 역방향 순회:
  bucket[3] → [1] 추가 → result: [1]
  bucket[2] → [2] 추가 → result: [1, 2]  ← k=2 달성!

결과: [1, 2] ✅
```

---

### 올바른 풀이

```ts
function topKFrequent(nums: number[], k: number): number[] {
  // Step 1. 빈도수 Map
  const count = new Map<number, number>();
  for (const num of nums) {
    count.set(num, (count.get(num) || 0) + 1);
  }

  // Step 2. 버킷 배열 (인덱스 = 빈도수)
  const bucket: number[][] = Array.from({ length: nums.length + 1 }, () => []);
  for (const [num, freq] of count) {
    bucket[freq].push(num);
  }

  // Step 3. 역방향 순회하며 k개 수집
  const result: number[] = [];
  for (let i = bucket.length - 1; i >= 0 && result.length < k; i--) {
    result.push(...bucket[i]);
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 빈도수 카운팅 + 버킷 채우기 + 역방향 순회 |
| 공간 | O(n) — Map + 버킷 배열 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- `getCountObj()`로 빈도수를 구하는 1단계 아이디어는 정확
- `count[num] = (count[num] || 0) + 1` 패턴 올바르게 작성

**보완할 점**
- 빈도수를 구한 후 **"어떻게 k개를 뽑는가"** 까지 연결하는 것이 이 문제의 핵심
- 버킷 배열 아이디어: **인덱스 = 빈도수**, 역방향으로 k개 수집
- `returnArr.length = k`는 미리 크기를 잡는 게 아님 → 그냥 `[]`로 시작하고 `push`

## 2번. Longest Consecutive Sequence — Medium · Hash Set

🔗 https://leetcode.com/problems/longest-consecutive-sequence/

### 문제 요구사항

- 정렬되지 않은 정수 배열 `nums`에서
- **연속된 숫자의 가장 긴 시퀀스 길이** 반환
- **시간복잡도 O(n)** 으로 풀어야 함

---

### 내가 작성한 코드 (오답)

```ts
function longestConsecutive(nums: number[]): number {
    const tempArray = [];
    let minValue = Math.min(...nums); // ❌ 전체 최솟값으로 시작

    tempArray.push(minValue);

    for(let i = 0; i < nums.length; i++){
        if(nums[i] === minValue + 1) {   // ❌ 배열 순서대로 탐색
            tempArray.push(minValue + 1);
            minValue += 1
        } else{
            return tempArray.length;     // ❌ 첫 번째 불일치에서 즉시 종료
        }
    }
};
```

---

### 틀린 이유

**❌ 버그 1: 배열이 정렬되어 있지 않아 `nums[i]` 순서 탐색이 무의미**

```
nums = [100, 4, 200, 1, 3, 2]

i=0: nums[0]=100, minValue=1 → 100 !== 2 → 즉시 return 1
실제 정답: 4 (1→2→3→4)

배열 순서대로 탐색하면 정렬되지 않은 배열에서 연속 시퀀스를 찾을 수 없음
```

**❌ 버그 2: 전체 최솟값에서만 시작 → 중간에 시작하는 시퀀스를 놓침**

```
nums = [10, 1, 2, 3, 4]

minValue = 1 → 1→2→3→4 (길이 4) → 정답
그러나 만약 최솟값부터 연속이 없다면?
nums = [5, 0, 3, 1, 2]
minValue = 0 → 0→1→2→3 (길이 4)
nums = [100, 200, 1, 2, 3]
minValue = 1 → 정답 3  (우연히 맞음)

그러나 단 하나의 최솟값만 시작점으로 삼으면
배열 내 여러 시퀀스 중 가장 긴 것을 찾을 수 없음
```

**❌ 버그 3: `else { return }` — 첫 불일치에서 즉시 종료**

```
연속이 끊겨도 다른 시퀀스가 더 길 수 있음
→ 첫 번째 불일치에서 종료하면 전체 탐색 불가
```

---

### 🧩 핵심 접근법 — Hash Set (O(n))

**왜 Hash Set인가?**

Set에 모든 숫자를 넣으면 특정 숫자의 존재 여부를 O(1)에 확인할 수 있다.
→ 정렬(O(n log n)) 없이 O(n)에 연속 시퀀스 탐색 가능.

**핵심 아이디어: 시작점 판별**

```
num - 1이 Set에 없는 숫자만 시작점으로 탐색
→ 중간부터 시작하면 이미 더 긴 시퀀스의 일부이므로 중복 탐색 방지
```

**접근 순서**

```
Step 1. Set 생성: Set(nums)
Step 2. 각 num에 대해:
    - set.has(num - 1)이 없으면 → 이 num이 시퀀스의 시작점
    - 시작점에서 num+1, num+2, ... 이 Set에 있는 동안 길이 증가
Step 3. 각 시작점에서 구한 길이의 최댓값 반환
```

**핵심 아이디어 시각화**

```
nums = [100, 4, 200, 1, 3, 2]
set  = {100, 4, 200, 1, 3, 2}

100: set.has(99)?  No  → 시작점! → 100, 101? No → 길이 1
4:   set.has(3)?   Yes → 시작점 아님 (skip)
200: set.has(199)? No  → 시작점! → 200, 201? No → 길이 1
1:   set.has(0)?   No  → 시작점! → 1→2→3→4→5? No → 길이 4 ✅
3:   set.has(2)?   Yes → 시작점 아님 (skip)
2:   set.has(1)?   Yes → 시작점 아님 (skip)

최댓값: 4 ✅
```

---

### 올바른 풀이

```ts
function longestConsecutive(nums: number[]): number {
  const set = new Set(nums); // O(n)
  let maxLen = 0;

  for (const num of set) {
    if (!set.has(num - 1)) {       // 시작점 판별
      let cur = num;
      let len = 1;

      while (set.has(cur + 1)) {   // 연속 탐색
        cur++;
        len++;
      }

      maxLen = Math.max(maxLen, len);
    }
  }

  return maxLen;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 각 숫자는 시작점 탐색 1회 + while 내부 1회만 처리 |
| 공간 | O(n) — Set 크기 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "최솟값부터 시작해서 +1씩 탐색" 방향성 자체는 연속 시퀀스 핵심 발상과 유사
- `minValue += 1`로 다음 연속값을 추적하는 아이디어는 올바름

**보완할 점**
- 정렬되지 않은 배열에서 **배열 인덱스 순서로 탐색하면 안 됨** → Set으로 값 존재 여부를 O(1)에 확인
- **단 하나의 시작점(전체 최솟값)이 아니라** `num - 1`이 없는 모든 숫자를 시작점으로 탐색
- 첫 불일치에서 `return`하지 말고 `maxLen`을 **갱신하면서 끝까지 탐색**

### 보완할 점 🔧

1. **`tempObj[char]`처럼 키를 지정해서 접근**

   ```js
   tempObj += 1;
   ```

   ```js
   for (...) {
     ...
     return obj; // ❌ 1회 반복 후 종료
   }

   for (...) { ... }
   return obj; // ✅
   ```

3. **빈 `return`은 `undefined` → 반환값 명시 필수**
   ```js
   return               // ❌ undefined
   return true / false  // ✅
   ```

4. **Bucket Sort 핵심 공식**
   ```js
   // 인덱스 = 빈도수인 버킷 배열

   for (const [num, freq] of count) bucket[freq].push(num);

   // 역방향으로 k개 수집

   ```

5. **Hash Set 시작점 판별 공식**
   ```js
   if (!set.has(num - 1)) {
     // 이 num이 시퀀스의 시작점
     // → num-1이 없다 = 더 긴 시퀀스의 일부가 아님
   }
   ```
