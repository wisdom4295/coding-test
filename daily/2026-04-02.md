## 1번. Longest Repeating Character Replacement — Medium · Sliding Window

🔗 https://leetcode.com/problems/longest-repeating-character-replacement/

### 문제 요구사항

- 문자열 `s`에서 최대 `k`개의 문자를 다른 문자로 바꿀 수 있을 때
- 같은 문자로만 이루어진 **가장 긴 구간의 길이** 반환
- 예: `s = "AABABBA"`, `k = 1` → `4`

---

### 🧩 핵심 접근법 — 가변 윈도우 (O(n))

**핵심 아이디어 딱 하나**

```
(윈도우 크기) - (윈도우 안 가장 많은 문자 수) <= k
→ 유효한 윈도우 (k번 바꾸면 전부 같은 문자로 만들 수 있음)
→ 조건 초과 시 left++ 로 윈도우 축소
```

**왜 이 공식인가?**

```
윈도우 안에 문자가 10개 있고 가장 많은 문자가 7개라면
→ 나머지 3개만 바꾸면 전부 같은 문자가 됨
→ 3 <= k 이면 유효한 윈도우
```

**시각화**

```
s = "AABABBA", k = 1

window  | 최다문자 | 크기-최다 | k=1 이하?
"A"     | A:1     | 1-1=0    | ✅
"AA"    | A:2     | 2-2=0    | ✅
"AAB"   | A:2     | 3-2=1    | ✅
"AABA"  | A:3     | 4-3=1    | ✅  maxLen=4
"AABAB" | A:3     | 5-3=2    | ❌ → left++
"ABAB"  | A:2     | 4-2=2    | ❌ → left++
"BAB"   | B:2     | 3-2=1    | ✅
"BABB"  | B:3     | 4-3=1    | ✅  maxLen=4
"BABBA" | B:3     | 5-3=2    | ❌ → left++
"ABBA"  | B:2     | 4-2=2    | ❌ → left++
"BBA"   | B:2     | 3-2=1    | ✅
"BBAB"  | B:3     | 4-3=1    | ✅  maxLen=4

결과: 4
```

---

### 올바른 풀이

```ts
function characterReplacement(s: string, k: number): number {
  const count = new Array(26).fill(0);
  const a = 'A'.charCodeAt(0);
  let left = 0;
  let maxFreq = 0;
  let maxLen = 0;

  for (let right = 0; right < s.length; right++) {
    count[s.charCodeAt(right) - a]++;
    maxFreq = Math.max(maxFreq, count[s.charCodeAt(right) - a]);

    // 유효하지 않은 윈도우 → left 축소
    while ((right - left + 1) - maxFreq > k) {
      count[s.charCodeAt(left) - a]--;
      left++;
    }

    maxLen = Math.max(maxLen, right - left + 1);
  }

  return maxLen;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(1) — 알파벳 26개 배열 |

---

### 핵심 공식 암기

```
유효 조건: (right - left + 1) - maxFreq <= k
초과 시:   left++ 로 윈도우 축소
```

## 2번. Minimum Size Subarray Sum — Medium · Sliding Window

🔗 https://leetcode.com/problems/minimum-size-subarray-sum/

### 문제 요구사항

- 양수 정수 배열 `nums`와 `target`이 주어졌을 때
- 합이 `target` **이상**이 되는 **가장 짧은 연속 구간의 길이** 반환
- 없으면 `0` 반환
- 예: `nums = [2,3,1,2,4,3]`, `target = 7` → `2` ([4,3])

---

### 🧩 핵심 접근법 — 가변 윈도우 축소 (O(n))

**핵심 아이디어 딱 하나**

```
sum >= target 되는 순간 → 최소 길이 갱신 + left++ 로 줄여보기
sum <  target           → right++ 로 확장
```

**시각화**

```
nums = [2, 3, 1, 2, 4, 3], target = 7

right=0: sum=2  < 7  → 확장
right=1: sum=5  < 7  → 확장
right=2: sum=6  < 7  → 확장
right=3: sum=8  >= 7 → minLen=4, left++ → sum=6
right=4: sum=10 >= 7 → minLen=3, left++ → sum=7
         sum=7  >= 7 → minLen=2, left++ → sum=4
right=5: sum=7  >= 7 → minLen=2, left++ → sum=3

결과: 2  ([4, 3]) ✅
```

---

### 올바른 풀이

```ts
function minSubArrayLen(target: number, nums: number[]): number {
  let left = 0;
  let sum = 0;
  let minLen = Infinity;

  for (let right = 0; right < nums.length; right++) {
    sum += nums[right]; // 오른쪽 확장

    while (sum >= target) {
      minLen = Math.min(minLen, right - left + 1); // 최소 길이 갱신
      sum -= nums[left];                            // 왼쪽 축소
      left++;
    }
  }

  return minLen === Infinity ? 0 : minLen;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(1) |

---

### 핵심 공식 암기

```
sum >= target → minLen 갱신 → sum -= nums[left] → left++
못 찾으면    → 0 반환 (minLen === Infinity ? 0 : minLen)
```

### 패턴 & 시간복잡도 요약

| 문제 | 패턴 | 시간 | 공간 | 핵심 1줄 |

---

### Sliding Window 전체 패턴 정리

| 종류 | 언제 쓰나 | 핵심 조건 | 예시 문제 |
|------|---------|---------|---------|
| 가변 — 확장 | 조건 만족할 때까지 right++ | 중복 없으면 확장 | Longest Substring |
| 가변 — 축소 | 조건 만족하면 left++ | sum >= target | Min Size Subarray |
| 가변 — 유효 판단 | 조건 초과 시 left++ | 크기 - maxFreq <= k | Char Replacement |
| 고정 | 크기가 정해진 경우 | 윈도우 크기 = s1.length | Permutation in String |
