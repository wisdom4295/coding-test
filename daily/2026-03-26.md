## 1. Two Sum — Easy · Hash Map

🔗 https://leetcode.com/problems/two-sum/

### 문제 요구사항

`nums[i] + nums[j] === target`을 만족하는 **인덱스 `[i, j]`** 반환 (단, `i ≠ j`)

---

### 내가 작성한 코드 (미완성)

```ts
function twoSum(nums: number[], target: number): number[] {
};

// 반복문을 1번만 돌려서 2개의 합이 target 값이 되면 됨
// 이중 for문이 아니게 하고 싶었음
// 1개를 잡고 1개를 더했을 때 케이스를 어떻게 생각해야 할지 모르겠음
```

---

### 막힌 지점 분석

| 구분 | 내용 |
|------|------|
| ✅ 잘한 점 | "이중 for문을 피하고 싶다"는 방향성이 Hash Map O(n) 풀이의 핵심 발상과 정확히 일치 |
| ✅ 잘한 점 | "1개를 잡고 1개를 더한다" 발상은 `target - nums[i]`를 찾는 Map 패턴과 동일 |
| ❌ 막힌 부분 | 발상에서 구현으로 이어지지 못함. 첫 줄(`complement = target - nums[i]`)만 써도 방향이 잡혔을 것 |

---

### 🧩 핵심 접근법 — Hash Map (O(n))

**왜 이중 루프를 피할 수 있는가?**

이중 루프는 "모든 쌍을 직접 비교"하는 방식 → O(n²).
Hash Map을 쓰면 "내가 지금 보는 숫자와 짝이 되는 숫자가 이미 나왔는지"를 O(1)에 확인할 수 있다.

**접근 순서**

```
Step 1. 현재 숫자 nums[i]를 본다
Step 2. 짝이 되는 값 = complement = target - nums[i]
Step 3. Map에 complement가 있으면 → 찾음! [map.get(complement), i] 반환
Step 4. 없으면 → 현재 값을 Map에 저장 (나중에 다른 수의 짝으로 쓰임)
```

**핵심 1줄 공식**

```
complement = target - nums[i]
→ map.has(complement) ? 찾음 : map.set(nums[i], i)
```

---

### 올바른 풀이

```ts
function twoSum(nums: number[], target: number): number[] {
    const map = new Map<number, number>(); // { 값: 인덱스 }

    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i]; // 나와 짝이 되는 값

        if (map.has(complement)) {
            return [map.get(complement)!, i]; // 짝을 찾으면 즉시 반환
        }

        map.set(nums[i], i); // 아직 못 찾았으면 현재 값 저장
    }
    return [];
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(n) — Map 크기 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "이중 루프를 피하고 싶다"는 의식이 이미 Hash Map 패턴의 핵심 발상과 일치
- 문제 의도(시간복잡도 최적화)를 빠르게 눈치챈 것

**보완할 점**
- 막혔을 때 `complement = target - nums[i]` 한 줄을 먼저 쓰고, "이걸 어떻게 찾지?" → Map으로 이어지는 흐름 연습
- Two Sum 3가지 풀이 시간복잡도 비교:

| 방식 | 시간 | 공간 | 비고 |
|------|------|------|------|
| Brute Force (이중 루프) | O(n²) | O(1) | 가장 직관적이지만 느림 |
| 정렬 + Two Pointers | O(n log n) | O(1) | 인덱스가 바뀌므로 이 문제엔 부적합 |
| **Hash Map (1회 순회)** | **O(n)** | **O(n)** | **이 문제의 의도형 정답** |

## 2. Best Time to Buy and Sell Stock — Easy · Array

🔗 https://leetcode.com/problems/best-time-to-buy-and-sell-stock/

### 문제 요구사항

- `prices[i]` = i번째 날 주식 가격
- 딱 1번 사고 1번 팔 수 있음
- **사는 날(i)이 파는 날(j)보다 반드시 앞서야 함 (i < j)**
- 최대 이익 반환 (이익이 없으면 0)

---

### 내가 작성한 코드 (논리 오류)

```ts
function maxProfit(prices: number[]): number {
    if (prices.length < 2) {
        throw new Error("배열에는 최소 2개의 요소가 필요합니다.");
    }

    let sellingPrice = prices; // ❌ 변수 이름과 역할이 반대
    let buyPrice = prices;     // ❌ 변수 이름과 역할이 반대[1]

    for (let i = 1; i < prices.length; i++) {
        if (prices[i] < sellingPrice) sellingPrice = prices[i]; // ← 실제로는 minPrice 역할
        if (prices[i] > buyPrice) buyPrice = prices[i];         // ← 실제로는 maxPrice 역할
    }

    return buyPrice - sellingPrice; // ❌ 순서(i < j) 조건 무시됨
}
```

---

### 틀린 이유

**❌ 버그 1: 변수 이름과 역할이 뒤바뀜**

```
sellingPrice → 실제로는 최솟값(구매가)을 저장하는 역할
buyPrice     → 실제로는 최댓값을 저장하는 역할
이름이 로직과 반대라서 코드를 짜다가 방향을 잃는 원인이 됨
```

**❌ 버그 2: 순서 제약(i < j) 무시**

예시: `[7, 1, 5, 3, 6]`

```
전체 min = 1 (인덱스 1)
전체 max = 7 (인덱스 0)
결과: 7 - 1 = 6

그러나 7은 인덱스 0, 1은 인덱스 1
→ 1에 사고 7에 파는 것은 불가능 (순서가 반대)
```

**❌ 버그 3: 이익이 음수일 때 0 반환 처리 없음**

문제 조건: 이익이 없으면 반드시 0을 반환해야 함.

---

### 🧩 핵심 접근법 — 최솟값 추적 (O(n))

**왜 "i < j" 순서를 자동으로 보장할 수 있는가?**

왼쪽에서 오른쪽으로 순회하면서 `minPrice`를 갱신하면,
`minPrice`는 항상 **현재 인덱스보다 앞에 있는 최솟값**이 된다.
따라서 `profit = price - minPrice`는 항상 "앞에서 사고 지금 파는" 이익이다.

**접근 순서**

```
Step 1. minPrice = Infinity, maxProfit = 0 으로 초기화
Step 2. 왼쪽부터 price를 하나씩 본다
Step 3. price < minPrice 이면 → minPrice 갱신 (더 싸게 살 수 있음)
Step 4. profit = price - minPrice → maxProfit보다 크면 갱신
Step 5. 끝까지 반복 후 maxProfit 반환
```

**핵심 아이디어 시각화**

```
prices =[2][3][4][5][6][1]

i=0: price=7, minPrice=7, profit=0, maxProfit=0
i=1: price=1, minPrice=1, profit=0, maxProfit=0  ← 더 싸게 살 수 있음
i=2: price=5, minPrice=1, profit=4, maxProfit=4
i=3: price=3, minPrice=1, profit=2, maxProfit=4
i=4: price=6, minPrice=1, profit=5, maxProfit=5  ← 갱신
i=5: price=4, minPrice=1, profit=3, maxProfit=5

결과: 5
```

---

### 올바른 풀이

```ts
function maxProfit(prices: number[]): number {
    let minPrice = Infinity; // 지금까지 본 가격 중 최솟값 (구매 기준)
    let maxProfit = 0;       // 지금까지 계산한 최대 이익

    for (const price of prices) {
        if (price < minPrice) {
            minPrice = price;            // 최솟값 갱신
        }
        const profit = price - minPrice; // 오늘 팔면 이익이 얼마인가
        if (profit > maxProfit) {
            maxProfit = profit;          // 최대 이익 갱신
        }
    }

    return maxProfit; // 이익 없으면 0 그대로 반환
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(1) — 변수 2개만 사용 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "판매가 - 구매가" 공식을 정확히 이해
- 반복문 구조 자체는 올바르게 설계됨
- `throw new Error` 처리 → 실제 문제에는 필요 없지만 방어적 코딩 의식이 좋은 습관

**보완할 점**
- 변수 이름을 역할에 맞게 먼저 짓기: `minPrice`, `maxProfit`
- "순서 제약(i < j)"이 있는 문제에서 Brute Force 먼저(`for j = i+1`), 이후 1회 순회로 최적화하는 습관
- 음수 이익 처리: `maxProfit = 0`으로 초기화하면 자동으로 해결됨

## 3. Valid Parentheses — Easy · Stack

🔗 https://leetcode.com/problems/valid-parentheses/

### 문제 요구사항

괄호 문자열 `s`가 올바른지 판단:
1. 열린 괄호는 같은 타입의 닫힌 괄호로 닫혀야 함
2. 닫힌 괄호는 가장 최근에 열린 괄호와 짝이어야 함
3. 모든 열린 괄호는 반드시 닫혀야 함

---

### 내가 작성한 코드 (오답)

```ts
function isValid(s: string): boolean {
    const stack = [];

    for (i = 0; i < s.length; i++) { // ❌ let i 빠짐 → 전역 변수 오염
        if (s[i] === '(' && s[i+1] === ')'   // ❌ i+1 기반 짝 검사
            || s[i] === '{' && s[i+1] === '}'
            || !s[i] === '[' && s[i+1] === ']' // ❌ !s[i] 연산자 오류
        ) {
            stack.push(s[i]);
        }
    }

    return Boolean(stack.length); // ❌ 스택에 뭔가 있으면 true → 완전히 반대
}
```

---

### 틀린 이유

**❌ 버그 1: `i+1` 기반 짝 검사 → 중첩 괄호 처리 불가**

```
"({[]})" 입력 시:
i=0: '(' → i+1이 '{' → 짝 아님 → push 안 됨
i=1: '{' → i+1이 '[' → 짝 아님 → push 안 됨
→ 스택에 아무것도 안 쌓임 → false 반환
올바른 답은 true ❌
```

스택의 역할은 "지금까지 열린 괄호를 기억"하는 것.
`i+1`만 보면 바로 옆이 짝인지만 판단 → 중첩 괄호 처리 불가.

**❌ 버그 2: `!s[i] === '['` 연산자 오류**

```ts
!s[i] === '['
// !s[i]는 문자열이 비어있지 않으면 항상 false
// false === '[' → 항상 false
// 의도한 건: s[i] === '['
```

**❌ 버그 3: `return Boolean(stack.length)` → 의미가 반대**

```
스택에 뭔가 남아있다 = 짝 못 맞춘 괄호가 있다 = invalid → false 반환해야 함
스택이 비어있다     = 모든 짝이 맞았다          = valid   → true 반환해야 함

올바른 반환: return stack.length === 0
```

**❌ 버그 4: `for (i = ...)` → `let i` 빠짐**

```ts
// 틀린 코드
for (i = 0; ...)  // i가 암묵적 전역 변수

// 올바른 코드
for (let i = 0; ...)
```

---

### 🧩 핵심 접근법 — Stack (O(n))

**왜 Stack이어야 하는가?**

괄호는 "가장 최근에 열린 것이 가장 먼저 닫혀야" 한다 → **LIFO(Last In, First Out)** 구조 → Stack.

**접근 순서**

```
Step 1. stack = [] 초기화
Step 2. 문자열을 왼쪽부터 순회
    - 열린 괄호 ('(', '{', '[') → stack.push(char)
    - 닫힌 괄호 (')', '}', ']') →
        a. 스택이 비어있으면 → return false (짝 없음)
        b. 스택 top이 mapping[char]과 다르면 → return false (짝 불일치)
        c. 일치하면 → stack.pop()
Step 3. 끝까지 돌고 난 후 → return stack.length === 0
```

**mapping 구조**

```ts
const mapping = {
    ')': '(',
    '}': '{',
    ']': '[',
};
// "닫힌 괄호가 왔을 때, 대응하는 열린 괄호가 스택 top에 있는가?"
```

**핵심 아이디어 시각화**

```
s = "({[]})"

char='(' → 열린 괄호 → push  → stack: ['(']
char='{' → 열린 괄호 → push  → stack: ['(', '{']
char='[' → 열린 괄호 → push  → stack: ['(', '{', '[']
char=']' → top '[' === mapping[']'] '['  → pop → stack: ['(', '{']
char='}' → top '{' === mapping['}'] '{'  → pop → stack: ['(']
char=')' → top '(' === mapping[')'] '('  → pop → stack: []

stack.length === 0 → true ✅
```

---

### 올바른 풀이

```ts
function isValid(s: string): boolean {
    const stack: string[] = [];
    const mapping: Record<string, string> = {
        ')': '(',
        '}': '{',
        ']': '[',
    };

    for (const char of s) {
        if (char === '(' || char === '{' || char === '[') {
            stack.push(char); // 열린 괄호 → 스택에 저장
        } else {
            // 닫힌 괄호 → 스택 top과 짝이 맞는지 확인
            if (stack.length === 0 || stack[stack.length - 1] !== mapping[char]) {
                return false;
            }
            stack.pop(); // 짝이 맞으면 스택에서 제거
        }
    }

    return stack.length === 0; // 모든 짝이 맞으면 스택이 비어있음
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(n) — 스택 최대 n/2 크기 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "스택을 사용해야 한다"는 방향성을 바로 잡음
- for 순회 구조를 바로 잡은 점

**보완할 점**
- 스택 패턴 3줄 공식 반드시 외우기:
  ```
  push(열린 괄호)
  pop(닫힌 괄호) + mapping 비교
  return stack.length === 0
  ```
- `i+1` 기반 검사는 스택 문제에서 쓰면 안 됨. 스택은 "현재"와 "과거(스택 top)"를 비교하는 구조
- `Boolean(stack.length)` vs `stack.length === 0` 차이 꼭 기억

## 4. Valid Palindrome — Easy · Two Pointers

🔗 https://leetcode.com/problems/valid-palindrome/

### 문제 요구사항

- 영숫자(알파벳 + 숫자)만 남기고
- 대소문자 구분 없이
- 앞뒤가 같으면 true

---

### 내가 작성한 코드 (대부분 맞지만 버그 있음)

```ts
function isOnlyString(s: string): string {
    if (typeof s !== 'string') return '';
    return s
        .replace(/\s/g, '')          // 공백 제거
        .replace(/[^A-Za-z]/g, '')   // ❌ 버그: 숫자도 제거됨
        .toUpperCase();
}

function isPalindrome(s: string): boolean {
    if (isOnlyString(s) === isOnlyString(s.split('').reverse().join(''))) return true;
    return false;
}
```

---

### 틀린 이유

**❌ 버그 1: `/[^A-Za-z]/g` → 숫자까지 제거됨**

```ts
// 틀린 정규식: 알파벳만 남김
.replace(/[^A-Za-z]/g, '')

// 올바른 정규식: 알파벳 + 숫자 남김
.replace(/[^A-Za-z0-9]/g, '')

// 예시: "0P" 입력 시
// 틀린 정규식 결과: "P"
// 올바른 정규식 결과: "0P"
```

**⚠️ 개선 포인트: 불필요한 reverse 문자열 생성**

```ts
// 현재: O(n) 공간의 새 문자열 생성
s.split('').reverse().join('')

// 개선: Two Pointers로 O(1) 공간에서 직접 비교 가능
```

---

### 🧩 핵심 접근법 — Two Pointers (O(n), O(1))

**왜 Two Pointers인가?**

팰린드롬 = 앞에서 읽으나 뒤에서 읽으나 같은 문자열.
양끝에서 포인터를 하나씩 좁혀오면서 글자가 다를 때 즉시 false를 반환하면 O(n)에 처리 가능.

**방법 1: 전처리 후 Two Pointers**

```
Step 1. 전처리: 소문자 변환 + 영숫자만 남김 (/[^a-z0-9]/g)
Step 2. left = 0, right = clean.length - 1
Step 3. left < right인 동안:
    - clean[left] !== clean[right] → return false
    - left++, right--
Step 4. return true
```

**방법 2: 전처리 없이 직접 Two Pointers (O(1) 공간)**

```
Step 1. left = 0, right = s.length - 1
Step 2. left < right인 동안:
    - 영숫자가 아니면 skip (left++ 또는 right--)
    - s[left].toLowerCase() !== s[right].toLowerCase() → return false
    - left++, right--
Step 3. return true
```

**핵심 아이디어 시각화**

```
s = "A man, a plan, a canal: Panama"

전처리 → "amanaplanacanalpanama"

left=0  'a' ↔ right=19 'a' ✅
left=1  'm' ↔ right=18 'm' ✅
left=2  'a' ↔ right=17 'a' ✅
... 모두 일치 → true ✅
```

---

### 올바른 풀이

```ts
// 방법 1: 전처리 후 Two Pointers
function isPalindrome(s: string): boolean {
    const clean = s.toLowerCase().replace(/[^a-z0-9]/g, ''); // ← 숫자 포함!
    let left = 0;
    let right = clean.length - 1;

    while (left < right) {
        if (clean[left] !== clean[right]) return false;
        left++;
        right--;
    }
    return true;
}

// 방법 2: 전처리 없이 O(1) 공간
function isPalindrome(s: string): boolean {
    let left = 0;
    let right = s.length - 1;
    const isAlNum = (c: string) => /[a-z0-9]/.test(c.toLowerCase());

    while (left < right) {
        while (left < right && !isAlNum(s[left])) left++;   // 영숫자 아니면 skip
        while (left < right && !isAlNum(s[right])) right--; // 영숫자 아니면 skip
        if (s[left].toLowerCase() !== s[right].toLowerCase()) return false;
        left++;
        right--;
    }
    return true;
}
```

| 방법 | 시간 | 공간 | 비고 |
|------|------|------|------|
| 방법 1 (전처리 후 투포인터) | O(n) | O(n) | clean 문자열 생성 |
| **방법 2 (직접 투포인터)** | **O(n)** | **O(1)** | **추가 공간 없음, 더 효율적** |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- `isOnlyString` 함수를 분리 → 코드 가독성을 높이는 좋은 습관
- `.replace().toUpperCase()` 전처리 흐름을 정확히 이해
- `split → reverse → join` 로직을 정확히 구현

**보완할 점**
- 정규식에서 숫자 포함 잊지 않기: `/[^A-Za-z0-9]/g`
- `reverse()` 대신 Two Pointers 방식(방법 2)을 익혀두면 공간복잡도 O(1) 달성
- `if (...) return true; return false;` → `return (...)` 한 줄로 단축 가능

## 5. Binary Search — Easy · Binary Search

🔗 https://leetcode.com/problems/binary-search/

### 문제 요구사항

- 정렬된 배열 `nums`에서 `target`의 **인덱스 반환**
- 없으면 **-1 반환**

---

### 내가 작성한 코드 (오답)

```ts
function search(nums: number[], target: number): number {
    if (nums.findIndex(ele => ele === target)) return; // ❌ 버그 3개
    return -1;
}
```

---

### 틀린 이유

**❌ 버그 1: `findIndex` 반환값을 boolean처럼 사용**

```ts
// findIndex는 숫자(인덱스)를 반환
// → 찾으면: 0, 1, 2, ... (인덱스)
// → 못 찾으면: -1

if (nums.findIndex(...)) {
    // 인덱스가 0이면 if(0) → false → 조건 불만족
    // 즉, 첫 번째 요소가 target이면 틀린 결과!
}

// 올바른 사용:
const idx = nums.findIndex(ele => ele === target);
if (idx !== -1) return idx;
```

**❌ 버그 2: `return` 반환값 없음**

```ts
return; // undefined 반환
// 문제는 인덱스(숫자)를 요구하는데 undefined 반환 → 오답
```

**❌ 버그 3: `findIndex`는 O(n) 선형 탐색**

```
이 문제의 핵심: 정렬된 배열에서 O(log n) Binary Search를 직접 구현하는 것
findIndex를 쓰면 정렬 배열의 특성을 전혀 활용하지 못함
```

---

### 🧩 핵심 접근법 — Binary Search (O(log n))

**왜 O(log n)이 가능한가?**

정렬된 배열에서는 중간값(`mid`)을 보고 target이 왼쪽에 있는지 오른쪽에 있는지 즉시 알 수 있다.
→ 탐색 범위를 매번 절반으로 줄임 → O(log n).

**Binary Search 3줄 공식**

```
lo = 0, hi = nums.length - 1

while (lo <= hi):
    mid = (lo + hi) >> 1       // 탐색 범위의 중간
    nums[mid] === target → 찾음, 반환
    nums[mid] < target  → lo = mid + 1  (오른쪽 절반 탐색)
    nums[mid] > target  → hi = mid - 1  (왼쪽 절반 탐색)

return -1  // 못 찾음
```

**핵심 아이디어 시각화**

```
nums = [-1, 0, 3, 5, 9, 12], target = 9

lo=0, hi=5 → mid=2, nums=3  < 9 → lo=3[7]
lo=3, hi=5 → mid=4, nums=9 === 9 → return 4 ✅[6]

총 2번 비교 (n=6일 때 최대 log2(6) ≈ 3번)
```

**왜 `(lo + hi) >> 1`을 쓰는가?**

```ts
// 일반적인 방법
const mid = Math.floor((lo + hi) / 2);

// 비트 연산 (동일하지만 더 빠름)
const mid = (lo + hi) >> 1;

// 오버플로우 완전 방지 (큰 숫자에서 안전)
const mid = lo + Math.floor((hi - lo) / 2);
```

---

### 올바른 풀이

```ts
function search(nums: number[], target: number): number {
    let lo = 0;
    let hi = nums.length - 1;

    while (lo <= hi) {
        const mid = (lo + hi) >> 1; // 중간 인덱스

        if (nums[mid] === target) {
            return mid;              // 찾음
        } else if (nums[mid] < target) {
            lo = mid + 1;            // 오른쪽 절반 탐색
        } else {
            hi = mid - 1;            // 왼쪽 절반 탐색
        }
    }

    return -1; // 못 찾음
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(log n) — 매번 절반씩 범위 축소 |
| 공간 | O(1) — 변수 3개만 사용 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- `-1` 반환 케이스를 인지한 점
- JS 내장 메서드 활용을 시도한 점 → 실무 감각

**보완할 점**
- `findIndex`는 O(n) → Binary Search 문제에서는 직접 lo/hi/mid로 구현
- `if(findIndex(...))` → 인덱스 0이 falsy인 함정 주의. 항상 `idx !== -1`로 비교
- Binary Search 3줄 공식은 외울 때까지 반복:
  ```
  lo = 0, hi = nums.length - 1
  mid = (lo + hi) >> 1
  nums[mid] < target → lo = mid + 1
  nums[mid] > target → hi = mid - 1
  ```
