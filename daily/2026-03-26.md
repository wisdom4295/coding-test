올바른 풀이

ts
function twoSum(nums: number[], target: number): number[] {
    const map = new Map<number, number>(); // 값 → 인덱스

    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i]; // 나와 짝이 되는 값
        if (map.has(complement)) {
            return [map.get(complement)!, i]; // 이미 봤으면 바로 반환
        }
        map.set(nums[i], i); // 아직 못 찾았으면 저장
    }
    return [];
}
💡 1회 순회, O(n) 시간, O(n) 공간

<details> <summary><strong>2. Best Time to Buy and Sell Stock</strong> — Easy · Array</summary>
🔗 링크: https://leetcode.com/problems/best-time-to-buy-and-sell-stock/

문제 요구사항: 사는 날이 파는 날보다 앞서야 한다(i < j), 최대 이익 반환 (없으면 0)

내가 작성한 코드 (논리 오류)

ts
function maxProfit(prices: number[]): number {
    if (prices.length < 2) {
        throw new Error("배열에는 최소 2개의 요소가 필요합니다.");
    }

    let sellingPrice = prices[0];
    let buyPrice = prices[1];

    for (let i = 1; i < prices.length; i++) {
        if (prices[i] < sellingPrice) sellingPrice = prices[i];
        if (prices[i] > buyPrice) buyPrice = prices[i];
    }

    return buyPrice - sellingPrice;
}
틀린 이유

❌ 버그 1: 변수 이름과 역할이 뒤바뀜

sellingPrice가 실제로는 최솟값(= 구매가)을 저장

buyPrice가 실제로는 최댓값을 저장

이름이 로직과 반대 → 코드 읽다가 방향을 잃는 원인

❌ 버그 2: 순서 제약(i < j) 무시

[7, 1, 5, 3, 6]에서 min=1, max=7이면 7-1=6처럼 보이지만

7은 1보다 앞에 있으므로 "1에 사고 7에 파는 것"은 불가능

전체 min/max만 구하면 순서 조건을 만족할 수 없음

❌ 버그 3: 이익이 음수일 때 0 반환 처리 없음

핵심 패턴: 최솟값 추적 — 왼쪽에서 minPrice 유지하며 price - minPrice 최댓값 추적

올바른 풀이

ts
function maxProfit(prices: number[]): number {
    let minPrice = Infinity;
    let maxProfit = 0;

    for (const price of prices) {
        if (price < minPrice) minPrice = price;
        const profit = price - minPrice;
        if (profit > maxProfit) maxProfit = profit;
    }
    return maxProfit;
}
💡 minPrice를 왼쪽에서 유지하면 "i < j" 조건이 자동으로 보장됨. O(n) 시간, O(1) 공간

잘한 점 ✅

"판매가 - 구매가" 공식을 정확히 이해

반복문 구조 자체는 올바르게 설계

throw new Error 처리 → 실제 문제에는 필요 없지만 방어적 코딩 의식 좋음

보완할 점 🔧

변수 이름을 역할에 맞게: minPrice, maxProfit으로 먼저 이름 짓고 시작

"순서 제약(i < j)"이 있는 문제에서 Brute Force 먼저 쓰고(for j = i+1), 이후 1회 순회로 최적화하는 습관

<details> <summary><strong>3. Valid Parentheses</strong> — Easy · Stack</summary>
🔗 링크: https://leetcode.com/problems/valid-parentheses/

문제 요구사항: 괄호가 올바르게 열리고 닫혔는지 판단

내가 작성한 코드 (오답)

ts
function isValid(s: string): boolean {
    const stack = [];
    for (i = 0; i < s.length; i++) {
        if (s[i] === '(' && s[i+1] === ')'
            || s[i] === '{' && s[i+1] === '}'
            || !s[i] === '[' && s[i+1] === ']'
        ) {
            stack.push(s[i]);
        }
    }
    return Boolean(stack.length);
}
틀린 이유

❌ 버그 1: i+1 기반 짝 검사 → 중첩 괄호 ({[]}) 처리 불가
   → 스택의 역할은 "지금까지 열린 괄호를 기억"하는 것인데, i+1만 보면 의미 없음

❌ 버그 2: !s[i] === '[' 연산자 오류
   → !s[i]는 빈 문자열이 아니면 항상 false → 의도한 건 s[i] === '['

❌ 버그 3: return Boolean(stack.length)
   → 스택에 뭔가 있으면 true → 완전히 반대
   → 올바른 결과는 스택이 비어야 true (모든 짝이 맞은 것)

❌ 버그 4: for (i = ...) → let i 없음 → 암묵적 전역 변수 오염

핵심 패턴: Stack — 열린 괄호 push, 닫힌 괄호 pop 후 mapping 비교, 끝에 stack.length === 0

올바른 풀이

ts
function isValid(s: string): boolean {
    const stack: string[] = [];
    const mapping: Record<string, string> = { ')': '(', '}': '{', ']': '[' };

    for (const char of s) {
        if (char === '(' || char === '{' || char === '[') {
            stack.push(char);
        } else {
            if (stack.length === 0 || stack[stack.length - 1] !== mapping[char]) {
                return false;
            }
            stack.pop();
        }
    }
    return stack.length === 0;
}
💡 스택 = 열린 괄호 기억 장치. 닫힐 때 pop + 짝 비교. 끝에 stack 비어야 통과

잘한 점 ✅

"스택을 사용해야 한다"는 방향성을 바로 잡음

for 순회 구조를 바로 잡은 점

보완할 점 🔧

스택 패턴 공식: push(열린) + pop(닫힌) + return stack.length === 0 반드시 외우기

i+1 기반 검사 → 스택에서 절대 쓰면 안 됨

let i로 항상 명시적 선언

<details> <summary><strong>4. Valid Palindrome</strong> — Easy · Two Pointers</summary>
🔗 링크: https://leetcode.com/problems/valid-palindrome/

문제 요구사항: 영숫자만 남기고 대소문자 무시 후 앞뒤가 같으면 true

내가 작성한 코드 (대부분 맞지만 버그 있음)

ts
function isOnlyString(s: string): string {
    if (typeof s !== 'string') return '';
    return s
        .replace(/\s/g, '')
        .replace(/[^A-Za-z]/g, '')  // ← 버그: 숫자 제거됨
        .toUpperCase();
}

function isPalindrome(s: string): boolean {
    if (isOnlyString(s) === isOnlyString(s.split('').reverse().join(''))) return true;
    return false;
}
틀린 이유

❌ 버그 1: /[^A-Za-z]/g → 숫자도 제거됨
   → 문제 조건은 "알파벳 + 숫자"를 남겨야 함 → /[^A-Za-z0-9]/g가 맞음
   → "0P" 입력 시 숫자가 제거되어 잘못된 결과

⚠️ 개선 포인트: split → reverse → join 대신 Two Pointers 쓰면 O(1) 공간

핵심 패턴: 전처리(숫자 포함!) + Two Pointers 양끝 비교

올바른 풀이

ts
// 방법 1: 전처리 후 Two Pointers
function isPalindrome(s: string): boolean {
    const clean = s.toLowerCase().replace(/[^a-z0-9]/g, ''); // ← 숫자 포함!
    let left = 0, right = clean.length - 1;
    while (left < right) {
        if (clean[left] !== clean[right]) return false;
        left++; right--;
    }
    return true;
}

// 방법 2: 전처리 없이 O(1) 공간
function isPalindrome(s: string): boolean {
    let left = 0, right = s.length - 1;
    const isAlNum = (c: string) => /[a-z0-9]/.test(c.toLowerCase());
    while (left < right) {
        while (left < right && !isAlNum(s[left])) left++;
        while (left < right && !isAlNum(s[right])) right--;
        if (s[left].toLowerCase() !== s[right].toLowerCase()) return false;
        left++; right--;
    }
    return true;
}
💡 방법 2가 추가 문자열 생성 없이 O(1) 공간으로 더 효율적

잘한 점 ✅

isOnlyString 함수 분리 → 코드 가독성 좋은 습관

split → reverse → join 로직 정확히 구현

전처리 흐름(replace → toUpperCase) 정확히 이해

보완할 점 🔧

정규식에서 숫자(0-9) 포함 잊지 않기: /[^A-Za-z0-9]/g

Two Pointers 방식(방법 2)을 익혀두면 공간복잡도 O(1) 달성

<details> <summary><strong>5. Binary Search</strong> — Easy · Binary Search</summary>
🔗 링크: https://leetcode.com/problems/binary-search/

문제 요구사항: 정렬된 배열에서 target 인덱스 반환, 없으면 -1

내가 작성한 코드 (오답)

ts
function search(nums: number[], target: number): number {
    if (nums.findIndex(ele => ele === target)) return;
    return -1;
}
틀린 이유

❌ 버그 1: findIndex 반환값을 boolean처럼 사용
   → 인덱스가 0이면 if(0) → false → 첫 번째 요소가 target이면 -1 반환

❌ 버그 2: return 반환값 없음
   → undefined 반환. 문제는 인덱스 숫자를 요구

❌ 버그 3: findIndex는 O(n) 선형 탐색
   → 이 문제는 정렬 배열에서 O(log n) Binary Search를 구현하는 것이 핵심

핵심 패턴: Binary Search — lo / hi / mid, 범위 절반씩 축소, O(log n)

올바른 풀이

ts
function search(nums: number[], target: number): number {
    let lo = 0, hi = nums.length - 1;

    while (lo <= hi) {
        const mid = (lo + hi) >> 1;
        if (nums[mid] === target) return mid;
        else if (nums[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
💡 (lo + hi) >> 1은 나누기 2와 동일한 비트 연산, 오버플로우 방지 가능

잘한 점 ✅

-1 반환 케이스를 인지한 점

JS 내장 메서드 활용 시도 → 실무 감각

보완할 점 🔧

findIndex는 O(n) → Binary Search 문제에서 쓰면 안 됨

if(findIndex(...)) → 인덱스 0이 falsy인 것 주의. 올바른 사용: const idx = ...; if (idx !== -1) return idx;

Binary Search 3줄 공식 외우기:

mid = (lo + hi) >> 1
nums[mid] < target → lo = mid + 1
nums[mid] > target → hi = mid - 1
