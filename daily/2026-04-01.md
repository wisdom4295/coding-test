### 🧩 핵심 접근법 — 정렬 + Two Pointers (O(n²))

**3줄 공식 — 이것만 외우면 됨**

```
Step 1. 정렬
Step 2. i 고정 → left = i+1, right = 끝
Step 3. sum 비교 → 0이면 push, 작으면 left++, 크면 right--
```

**시각화**

```
nums = [-4, -1, -1, 0, 1, 2]  (정렬 후)

i=1: nums[i]=-1
  left=2(-1), right=5(2) → sum = -1-1+2 = 0 ✅ push([-1,-1,2])
  left=3(0),  right=4(1) → sum = -1+0+1 = 0 ✅ push([-1,0,1])

결과: [[-1,-1,2], [-1,0,1]]
```

---

### 올바른 풀이

```ts
function threeSum(nums: number[]): number[][] {
  nums.sort((a, b) => a - b);
  const result: number[][] = [];

  for (let i = 0; i < nums.length - 2; i++) {
    if (nums[i] > 0) break;
    if (i > 0 && nums[i] === nums[i - 1]) continue; // i 중복 스킵

    let left = i + 1;
    let right = nums.length - 1;

    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];

      if (sum === 0) {
        result.push([nums[i], nums[left], nums[right]]);
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;
        left++;
        right--;
      } else if (sum < 0) {
        left++;
      } else {
        right--;
      }
    }
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n²) |
| 공간 | O(1) |

### 🧩 핵심 접근법 — Set + 가변 윈도우 (O(n))

**왜 Sliding Window인가?**

"중복 없는 연속 구간"을 찾을 때 → 구간의 왼쪽/오른쪽을 조절하는 Sliding Window가 최적.
모든 쌍을 비교하는 Brute Force는 O(n³) → Sliding Window로 O(n)으로 줄임.

**접근 순서**

```
Step 1. Set(현재 윈도우 안의 문자들), left=0, maxLen=0 초기화
Step 2. right를 0부터 끝까지 이동:
    - s[right]가 Set에 없으면 → Set에 추가, maxLen 갱신
    - s[right]가 Set에 있으면 → s[left]를 Set에서 제거 + left++
      (중복이 사라질 때까지 반복)
Step 3. maxLen 반환
```

**핵심 아이디어 시각화**

```
s = "abcabcbb"

right=0: 'a' 없음 → Set{a}        window="a"    maxLen=1
right=1: 'b' 없음 → Set{a,b}      window="ab"   maxLen=2
right=2: 'c' 없음 → Set{a,b,c}    window="abc"  maxLen=3
right=3: 'a' 있음 → 'a' 제거, left++ → Set{b,c,a}  window="bca"  maxLen=3
right=4: 'b' 있음 → 'b' 제거, left++ → Set{c,a,b}  window="cab"  maxLen=3
right=5: 'c' 있음 → 'c' 제거, left++ → Set{a,b,c}  window="abc"  maxLen=3
right=6: 'b' 있음 → 'a' 제거 → 'b' 제거 → Set{c,b} window="cb"  maxLen=3
right=7: 'b' 있음 → 'c' 제거 → 'b' 제거 → Set{b}   window="b"   maxLen=3

결과: 3 ✅
```

---

### 올바른 풀이

```ts
function lengthOfLongestSubstring(s: string): number {
  const set = new Set<string>();
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < s.length; right++) {
    while (set.has(s[right])) {
      set.delete(s[left]);
      left++;
    }

    set.add(s[right]);
    maxLen = Math.max(maxLen, right - left + 1);
  }

  return maxLen;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — right, left 각 최대 n번 이동 |
| 공간 | O(min(n, 26)) — Set 크기 |

---

## 2번. Permutation in String — Medium · Sliding Window

🔗 https://leetcode.com/problems/permutation-in-string/

### 문제 요구사항

- `s1`의 순열(애너그램)이 `s2` 안에 포함되면 `true`
- 예: `s1="ab"`, `s2="eidbaooo"` → `true` ("ba"가 포함)

---

### 내가 작성한 코드 (오답)

```js
function checkInclusion(s1, s2) {
    if(s2.includes(s1)
    || s2.split("").reverse().join("").includes(s1)) return true;

  return false;
};
```

---

### 틀린 이유

**❌ 버그 1: `s2.includes(s1)` — 순열이 아니라 s1 자체를 찾음**

```
s1 = "ab", s2 = "eidbaooo"

s2.includes("ab") → false  (s2에 "ab"가 없음)
정답은 true ("ba"가 s2에 포함되어 있으므로)

includes()는 "정확히 같은 문자열"을 찾음
순열(같은 문자들의 다른 배열)을 찾는 게 아님
```

**❌ 버그 2: `reverse()`로는 순열의 1가지만 처리**

```
s1 = "abc"의 순열: "abc", "acb", "bac", "bca", "cab", "cba" → 6가지
reverse()는 그 중 "cba" 1가지만 만들어냄 → 나머지 4가지를 놓침
```

**❌ 버그 3: 접근 방식 자체가 틀림**

```
순열 포함 여부 = "s1과 같은 문자 구성을 가진 연속 구간이 s2에 있는가"
→ s1.length 크기의 고정 윈도우를 s2 위에서 슬라이드하며 빈도수 비교
```

---

### 🧩 핵심 접근법 — 고정 크기 윈도우 + 빈도수 비교 (O(n))

**왜 고정 크기 윈도우인가?**

순열은 길이가 같고 문자 구성만 다름.
→ `s1.length` 크기의 윈도우를 s2 위에서 한 칸씩 밀면서 빈도수가 같은지 확인.

**접근 순서**

```
Step 1. s1의 빈도수 배열(26개) 생성
Step 2. s2의 첫 s1.length 구간 빈도수 배열 생성
Step 3. 두 배열이 같으면 → true
Step 4. 윈도우를 한 칸 오른쪽으로:
    - 오른쪽 새 문자 빈도수 +1
    - 왼쪽 나간 문자 빈도수 -1
    - 다시 비교
Step 5. 끝까지 못 찾으면 → false
```

**핵심 아이디어 시각화**

```
s1 = "ab"  s2 = "eidbaooo"
s1 빈도: {a:1, b:1}

윈도우 크기 = 2

[e,i]: {e:1,i:1} ≠ {a:1,b:1} → false
[i,d]: {i:1,d:1} ≠ {a:1,b:1} → false
[d,b]: {d:1,b:1} ≠ {a:1,b:1} → false
[b,a]: {b:1,a:1} === {a:1,b:1} → true ✅
```

---

### 올바른 풀이

```ts
function checkInclusion(s1: string, s2: string): boolean {
  if (s1.length > s2.length) return false;

  const freq1 = new Array(26).fill(0);
  const freq2 = new Array(26).fill(0);
  const a = 'a'.charCodeAt(0);

  for (let i = 0; i < s1.length; i++) {
    freq1[s1.charCodeAt(i) - a]++;
    freq2[s2.charCodeAt(i) - a]++;
  }

  if (freq1.join() === freq2.join()) return true;

  for (let i = s1.length; i < s2.length; i++) {
    freq2[s2.charCodeAt(i) - a]++;              // 오른쪽 새 문자 추가
    freq2[s2.charCodeAt(i - s1.length) - a]--;  // 왼쪽 나간 문자 제거

    if (freq1.join() === freq2.join()) return true;
  }

  return false;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — s2 1회 순회 |
| 공간 | O(1) — 크기 26 배열 2개 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "s2 안에서 찾아야 한다"는 방향 맞음
- `includes()` 등 JS 내장 메서드 활용 의식 좋음

**보완할 점**
- `includes()`는 완전 일치만 찾음 → 순열은 빈도수 비교 필요
- 순열 포함 = **고정 크기 윈도우** + **빈도수 배열 비교** 패턴 암기:
  ```ts
  freq2[새 문자]++
  freq2[나간 문자]--
  ```
- 1번(가변 윈도우) vs 2번(고정 윈도우) 차이 명확히 구분하기
