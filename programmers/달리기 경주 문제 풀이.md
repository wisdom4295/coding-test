해시맵(Map) 완전 정복 노트
2026-04-08 | 달리기 경주 문제 풀이 과정 기록

1. 왜 Map을 쓰는가
문제 상황:

players 최대 50,000명

callings 최대 1,000,000번 반복

indexOf 는 매번 배열 전체 탐색 → O(n²) → 시간초과

해결:

Map 은 이름 입력하면 즉시 순위 반환 → O(1) → 통과

text
"배열에서 값을 이름으로 자주 찾아야 해" → Map
"indexOf 쓰면 느릴 것 같아"            → Map
2. Map 문법 4개
javascript
const map = new Map();     // 만들기
map.set("mumu", 0);        // 넣기 (key, value)
map.get("mumu");           // 꺼내기 → 0 반환
map.set("mumu", 2);        // 수정
3. map vs forEach 차이
javascript
// map: 반복 + 새 배열 반환
const result = [1,2,3].map(x => x * 2);
// result = [2, 4, 6]

// forEach: 반복만 (반환값 없음)
[1,2,3].forEach(x => x * 2);
// 아무것도 반환 안 함
text
결과 배열 필요해 → map
그냥 반복만 해   → forEach
4. 달리기 경주 흐름
text
players = ["mumu", "pepper", "olivia"]
callings = ["pepper"]

1단계: Map 초기 세팅
map = { "mumu"→0, "pepper"→1, "olivia"→2 }
rank = ["mumu", "pepper", "olivia"]

2단계: "pepper" 호출
  → map.get("pepper") = 1
  → 앞 선수 = rank[0] = "mumu"
  → 배열 교체: ["pepper", "mumu", "olivia"]
  → Map 업데이트: "pepper"→0, "mumu"→1
5. 최종 통과 코드
javascript
function solution(players, callings) {
  const rank = new Map();
  const solution = [...players];

  players.forEach((ele, idx) => rank.set(ele, idx));

  callings.forEach((ele) => {
    const index = rank.get(ele);
    const ahead = solution[index - 1];

    // 배열 교체
    [solution[index-1], solution[index]] = [solution[index], solution[index-1]];

    // Map 업데이트 (핵심!)
    rank.set(ele, index - 1);
    rank.set(ahead, index);
  });

  return solution;
}
6. ❌ 내가 틀렸던 오점
오점 1: Map 업데이트를 안 함
javascript
// ❌ 자리 교체만 하고 Map은 옛날 순위 그대로
[solution[index-1], solution[index]] = [solution[index], solution[index-1]];

// ✅ 교체 후 반드시 Map도 업데이트
rank.set(ele, index - 1);
rank.set(ahead, index);
오점 2: forEach 대신 map 사용
javascript
// ❌ 반환값 안 쓰는데 map 사용 → 낭비
callings.map((ele) => { ... })

// ✅ 반복만 할 때는 forEach
callings.forEach((ele) => { ... })
오점 3: indexOf로 시작 → 시간초과
javascript
// ❌ O(n²)
const index = solution.indexOf(ele);

// ✅ O(1)
const index = rank.get(ele);
