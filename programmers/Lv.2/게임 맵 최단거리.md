## 1. BFS가 뭔가

> 돌을 물에 던지면 파문이 퍼지듯, 시작점에서 **가까운 곳부터 동심원 모양으로 탐색**하는 알고리즘

- **BFS = Breadth-First Search = 너비 우선 탐색**
- 최단거리 문제에 사용 → 처음 도달했을 때가 무조건 최단거리
- DFS는 한 방향으로 끝까지 가서 최단거리 보장 안 됨 → BFS만 가능

---

## 2. 핵심 자료구조: 큐(Queue)

> 줄 서기랑 똑같다. **먼저 들어온 것이 먼저 나온다.**

```
앞 [ A, B, C ] 뒤

→ A가 먼저 들어왔으니 A부터 꺼냄
→ B는 A 끝날 때까지 대기
→ 새로운 항목은 뒤에 추가
```

### 큐 동작 2가지

| 동작 | 코드 | 설명 |
|------|------|------|
| 꺼내기 | `queue.shift()` | 앞에서 1개 꺼냄 |
| 추가하기 | `queue.push(값)` | 뒤에 붙임 |

## 3. r, c, nr, nc 가 뭔지

```
r  = 지금 내가 서 있는 행 (현재 row)
c  = 지금 내가 서 있는 열 (현재 col)
nr = 다음에 이동할 행 (next row)
nc = 다음에 이동할 열 (next col)
```

### 격자에서 위치 읽는 법

```
      열0  열1  열2
행0  [ 1,   1,   1 ]
행1  [ 1,   1,   1 ]
행2  [ 1,   1,   1 ]  ← 목적지: maps[행2][열2]
```

- **코드에서**: `maps[r][c]`
- **시작점**: `maps[0][0]`
- **목적지**: `maps[rows-1][cols-1]`

---

## 4. 4방향 이동 원리

내가 `(행1, 열1)` 에 있을 때:

```
          북: 행0, 열1  (r-1, c)
           ↑
서: 행1,열0 ← 나 → 동: 행1, 열2  (r, c+1)
           ↓
          남: 행2, 열1  (r+1, c)
```

### dirs 배열 의미

```javascript
const dirs = [[0,1], [0,-1], [1,0], [-1,0]];
//             동      서      남     북
//            [dr,dc] [dr,dc] [dr,dc] [dr,dc]
```

- `dr` = 행 변화량, `dc` = 열 변화량
- `nr = r + dr`, `nc = c + dc`
- **순서는 아무 상관 없음** → 최단거리 결과는 항상 동일

## 5. visited 배열

> 방문 기록 장부. 맵과 똑같은 크기의 2D 배열.

```javascript
const visited = Array.from({length: rows}, () => Array(cols).fill(false));
```

- 처음엔 전부 `false` (미방문)
- 큐에 추가할 때 `true` 로 바꿈
- **꺼낼 때가 아니라 넣을 때 체크** → 안 그러면 같은 칸이 큐에 중복으로 쌓임

### 단계별 visited 변화 (3×3 맵 예시)

```
시작 후:        1회전 후:       2회전 후:       3회전 후:
T  F  F         T  T  F         T  T  T         T  T  T
F  F  F    →    T  F  F    →    T  T  F    →    T  T  F
F  F  F         F  F  F         F  F  F         T  F  F
```

## 6. BFS 3가지 핵심 규칙

```
1. 큐에서 꺼낸다 (shift)
2. 4방향으로 nr, nc 계산
3. 조건 3가지 통과하면 → visited 체크하고 큐에 넣는다
```

### 조건 4가지 (모두 통과해야 추가)

| 조건 | 코드 | 의미 |
|------|------|------|
| 행 범위 | `nr >= 0 && nr < rows` | 격자 위아래 벗어나지 않음 |
| 열 범위 | `nc >= 0 && nc < cols` | 격자 좌우 벗어나지 않음 |
| 미방문 | `!visited[nr][nc]` | 아직 간 적 없음 |
| 길 있음 | `maps[nr][nc] !== 0` | 벽이 아님 |

---

## 7. dist(거리)가 뭔지

```
queue = [[행, 열, dist]]

dist = 시작점(0,0)에서 이 칸까지 이동한 칸 수
시작점 자체 = 1 (문제 규칙)
이웃 칸으로 이동할 때마다 +1
```

```
(행0,열0) → dist 1
(행0,열1) → dist 2
(행1,열1) → dist 3
...
목적지 처음 도달 시 dist = 최단거리
```

- **dist는 큐에 처음 넣을 때부터 같이 저장**
- `queue.shift()` 로 꺼낼 때 `const [r, c, dist] = queue.shift()` 로 분해

---

## 8. 완성 코드

```javascript
function solution(maps) {
  const rows = maps.length;
  const cols = maps[0].length;
  const dirs = [[0,1],[0,-1],[1,0],[-1,0]];

  const visited = Array.from({length: rows}, () => Array(cols).fill(false));
  const queue = [[0, 0, 1]];
  visited[0][0] = true;

  while (queue.length) {
    const [r, c, dist] = queue.shift();

    if (r === rows-1 && c === cols-1) return dist;

    for (const [dr, dc] of dirs) {
      const nr = r + dr;
      const nc = c + dc;

      if (
        nr >= 0 && nr < rows &&
        nc >= 0 && nc < cols &&
        !visited[nr][nc] &&
        maps[nr][nc] !== 0
      ) {
        visited[nr][nc] = true;
        queue.push([nr, nc, dist + 1]);
      }
    }
  }

  return -1;
}
```

## 9. ❌ 내가 틀렸던 오점 기록

### 오점 1: 동남쪽만 이동하면 된다고 생각함
```
❌ 틀린 생각: 목적지가 오른쪽 아래니까 동쪽/남쪽만 이동하면 됨
✅ 맞는 것: 4방향 모두 필요 → 벽을 돌아가야 할 수도 있음
```

### 오점 2: visited를 1D 배열로 만듦
```javascript
❌ const visited = new Array.from({lenth:maps.length}, (_, i)=>false);
   // 문제1: new 붙이면 안 됨
   // 문제2: length 오타 (lenth)
   // 문제3: 1D 배열 → 2D 배열이어야 함

✅ const visited = Array.from({length: rows}, () => Array(cols).fill(false));
```

### 오점 3: queue 시작값을 1D로 만듦
```javascript
❌ const queue = [0, 0, 1];      // 1D 배열
✅ const queue = [[0, 0, 1]];    // 2D 배열 (배열 안에 배열)
```

### 오점 4: visited 재선언에 const 붙임
```javascript
❌ const visited[0][0] = true;   // 이미 선언된 변수에 const 또 붙임
✅ visited[0][0] = true;         // 그냥 값만 바꾸면 됨
```

### 오점 5: 목적지 조건에 = 하나만 씀 (할당 vs 비교)
```javascript
❌ if(r === rows-1 && c = cols-1)    // c = cols-1 은 할당!
✅ if(r === rows-1 && c === cols-1)  // === 비교연산자 써야 함
```

### 오점 6: queue.push에 배열로 안 감쌈
```javascript
❌ queue.push(nr, nc, dist + 1)       // 3개 별도 인자로 push
✅ queue.push([nr, nc, dist + 1])     // 배열 하나로 감싸서 push
```

### 오점 7: 범위 체크 방식 혼동
```javascript
❌ rows > nr && cols > nc             // 최솟값 체크 누락
✅ nr >= 0 && nr < rows && nc >= 0 && nc < cols   // 상하한 모두 체크
```

### 오점 8: visited 체크를 nr,nc가 아닌 r,c로 함
```javascript
❌ !visited[r][c]    // 현재 위치 체크 (이미 방문한 곳)
✅ !visited[nr][nc]  // 다음 위치 체크 (가려는 곳)
```
