**❌ 버그 1: 정렬 비교 함수 미완성**

```ts
// ❌ 비교 함수가 비어있음
intervals.sort((a, b) => )

// ✅ start 기준 오름차순 정렬
intervals.sort((a, b) => a[0] - b[0])
//                         ↑ 첫번째 요소(start)를 기준으로 비교
```

**❌ 버그 2: 병합 로직 미구현**

```
핵심인 "겹치면 합치고, 안 겹치면 새로 추가" 로직이 없음
→ 정렬만 하고 아무것도 하지 않음
```

---

### 🧩 핵심 접근법 — 정렬 후 병합 (O(n log n))

**겹침 판단 공식**

```
두 구간 [a, b]와 [c, d]에서:
c <= b  → 겹침 → end = Math.max(b, d)
c >  b  → 안 겹침 → result에 새 구간 push
```

**접근 순서**

```
Step 1. start 기준 정렬: intervals.sort((a, b) => a[0] - b[0])
Step 2. 첫 구간을 result에 넣고 시작
Step 3. i = 1부터 순회:
    - cur[0] <= last[1] → 겹침 → last[1] = Math.max(last[1], cur[1])
    - cur[0] >  last[1] → 안 겹침 → result.push(cur)
Step 4. result 반환
```

---

### 올바른 풀이

```ts
function merge(intervals: number[][]): number[][] {
  intervals.sort((a, b) => a[0] - b[0]); // ✅ start 기준 정렬

  const result: number[][] = [intervals[0]];

  for (let i = 1; i < intervals.length; i++) {
    const cur = intervals[i];
    const last = result[result.length - 1];

    if (cur[0] <= last[1]) {
      last[1] = Math.max(last[1], cur[1]); // 겹침 → end 갱신
    } else {
      result.push(cur);                     // 안 겹침 → 새 구간 추가
    }
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n log n) — 정렬 비용 |
| 공간 | O(n) — result 배열 |

## 1번. 3Sum — Medium · Two Pointers

🔗 https://leetcode.com/problems/3sum/

### 문제 요구사항

- 정수 배열 `nums`에서 합이 0이 되는 **세 수의 조합** 반환
- 중복 조합 없이
- 예: `[-1, 0, 1, 2, -1, -4]` → `[[-1,-1,2],[-1,0,1]]`

---

### 내가 작성한 코드 (오답)

```ts
// -remainnNum = num[i] + num[j]
function threeSum(nums: number[]): number[][]{
    const result = [];

    for(let i = 0; i < nums.length; i++){
        for(let j = 1; j < nums.length; j++){        // ❌ j가 항상 1부터 시작
            if(nums[i]+ nums[j] === -(nums[i]+ nums[j])) { // ❌ 수식 오류
                result.push(nums[i])  // ❌ 세 수가 아니라 같은 값 3번 push
                result.push(nums[i])
                result.push(nums[i])
            }
        }
    }

    return result;
};
```

---

### 틀린 이유

**✅ 진전된 점 먼저**

```
"세 수의 합 = 0" → "나머지 한 수 = -(두 수의 합)" 발상은 정확!
remainNum = -(nums[i] + nums[j]) 방향성 맞음 ✅
```

**❌ 버그 1: `nums[i] + nums[j] === -(nums[i] + nums[j])` — 수식 오류**

```
nums[i] + nums[j] === -(nums[i] + nums[j])
→ x === -x 를 풀면 x = 0일 때만 성립
→ 두 수의 합이 0일 때만 true
→ 세 번째 수를 전혀 찾지 않음

올바른 조건: 세 번째 수 = -(nums[i] + nums[j])
→ 이걸 배열에서 찾아야 함
```

**❌ 버그 2: `j`가 항상 1부터 시작 → `i`와 `j`가 겹침**

```ts
for(let i = 0; i < nums.length; i++){
    for(let j = 1; j < nums.length; j++){  // j = 1 고정
        // i=1, j=1 이면 같은 인덱스 사용
    }
}

// ✅ j는 i+1부터 시작해야 함
for(let j = i + 1; ...)
```

**❌ 버그 3: `result.push(nums[i])`를 3번 → 같은 값만 3개 push**

```ts
result.push(nums[i])  // 첫 번째 수
result.push(nums[i])  // ← 똑같이 nums[i]
result.push(nums[i])  // ← 똑같이 nums[i]

// ✅ 세 수를 배열로 묶어서 push
result.push([nums[i], nums[j], thirdNum])
```

**❌ 버그 4: 중복 조합 처리 없음**

```
[-1, 0, 1, 2, -1, -4] 정렬 후 [-4, -1, -1, 0, 1, 2]
i=1, i=2 둘 다 -1인 경우 → 중복 결과 발생
→ 중복 스킵 로직 필요
```

**❌ 버그 5: 이중 for문 → O(n²) — 세 번째 수 찾기까지 O(n³)**

```
Brute Force 이중 루프로 두 수를 고르고 세 번째를 또 탐색하면 O(n³)
정렬 + Two Pointers를 쓰면 O(n²)로 줄일 수 있음
```

---

### 🧩 핵심 접근법 — 정렬 + Two Pointers (O(n²))

**왜 Two Pointers인가?**

정렬 후 `i`를 고정하면 나머지 두 수의 합 = `-nums[i]`를 찾는 문제가 된다.
정렬된 배열에서 두 수의 합을 찾는 건 양끝 포인터(Two Pointers)로 O(n)에 해결 가능.

**접근 순서**

```
Step 1. 정렬: nums.sort((a, b) => a - b)
Step 2. i = 0부터 nums.length - 2까지 순회
    - nums[i] > 0이면 → 세 수 합이 0 불가 → break
    - 중복 스킵: i > 0 && nums[i] === nums[i-1] → continue
Step 3. left = i+1, right = nums.length - 1
Step 4. left < right인 동안:
    - sum = nums[i] + nums[left] + nums[right]
    - sum === 0 → result.push([...]) → 중복 스킵하며 left++, right--
    - sum < 0   → left++ (합을 키워야 함)
    - sum > 0   → right-- (합을 줄여야 함)
```

**핵심 아이디어 시각화**

```
nums = [-1, 0, 1, 2, -1, -4]
정렬  → [-4, -1, -1, 0, 1, 2]

i=0: nums[i]=-4, left=1, right=5
  sum = -4 + (-1) + 2 = -3 < 0 → left++
  sum = -4 + (-1) + 2 = -3 < 0 → left++
  sum = -4 + 0 + 2 = -2 < 0 → left++
  sum = -4 + 1 + 2 = -1 < 0 → left++
  left >= right → 종료

i=1: nums[i]=-1, left=2, right=5
  sum = -1 + (-1) + 2 = 0 → ✅ push([-1,-1,2])
  left++, right--
  sum = -1 + 0 + 1 = 0 → ✅ push([-1,0,1])
  left++, right--
  left >= right → 종료

i=2: nums[i]=-1, nums[i]===nums[i-1] → continue (중복 스킵)

i=3: nums[i]=0, left=4, right=5
  sum = 0 + 1 + 2 = 3 > 0 → right--
  left >= right → 종료

결과: [[-1,-1,2], [-1,0,1]] ✅
```

---

### 올바른 풀이

```ts
function threeSum(nums: number[]): number[][] {
  nums.sort((a, b) => a - b); // Step 1. 정렬
  const result: number[][] = [];

  for (let i = 0; i < nums.length - 2; i++) {
    if (nums[i] > 0) break;                           // 최솟값이 양수면 합 불가
    if (i > 0 && nums[i] === nums[i - 1]) continue;  // i 중복 스킵

    let left = i + 1;
    let right = nums.length - 1;

    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];

      if (sum === 0) {
        result.push([nums[i], nums[left], nums[right]]);
        while (left < right && nums[left] === nums[left + 1]) left++;   // left 중복 스킵
        while (left < right && nums[right] === nums[right - 1]) right--; // right 중복 스킵
        left++;
        right--;
      } else if (sum < 0) {
        left++;   // 합을 키워야 함
      } else {
        right--;  // 합을 줄여야 함
      }
    }
  }

  return result;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n²) — 정렬 O(n log n) + 이중 루프 O(n²) |
| 공간 | O(1) — result 제외 추가 공간 없음 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "세 번째 수 = -(두 수의 합)" 핵심 발상 정확히 파악
- 이중 루프로 두 수를 고르려는 방향성 맞음

**보완할 점**
- `nums[i] + nums[j] === -(nums[i] + nums[j])` 는 `x === -x` → x=0 일 때만 성립. 세 번째 수를 **배열에서 찾는** 코드가 필요
- `j`는 `i+1`부터: 같은 인덱스 사용 방지
- `result.push([nums[i], nums[j], 세번째수])` 처럼 **배열로 묶어** push
- Two Pointers로 O(n²)까지 최적화: `left = i+1, right = n-1`에서 수렴
- 중복 스킵 공식: `if (i > 0 && nums[i] === nums[i-1]) continue`

## 2번. Container With Most Water — Medium · Two Pointers

🔗 https://leetcode.com/problems/container-with-most-water/

### 문제 요구사항

- `height[i]`는 i번째 막대의 높이
- 두 막대 사이에 담을 수 있는 **최대 물의 양** 반환
- 넓이 = `Math.min(height[left], height[right]) * (right - left)`

---

### 내가 작성한 코드 (미완성)

```ts
// 가로 * 세로 값이 제일 큰거를 구하면 되는 거잖아?
// 근데 구간을 어떻게 잡지...?
function maxArea(height: number[]): number {
    // 미구현
};
```

---

### 틀린 이유

**✅ 핵심 발상은 정확**

```
"가로 * 세로" = (right - left) * Math.min(height[left], height[right])
→ 완전히 맞는 발상! 공식 자체를 이미 이해하고 있음 ✅
```

**❌ 막힌 지점: "구간을 어떻게 잡지?"**

```
이 막힌 지점의 답이 바로 Two Pointers!

Brute Force: 모든 쌍을 비교 → O(n²)
Two Pointers: 양끝에서 좁혀오기 → O(n)

핵심 질문: "어떤 포인터를 이동해야 하는가?"
→ 짧은 쪽을 이동 (긴 쪽을 이동하면 넓이가 절대 안 커짐)
```

**왜 짧은 쪽을 이동해야 하는가?**

```
현재 넓이 = min(h[left], h[right]) * (right - left)

만약 h[left] < h[right] 일 때 right를 안으로 이동하면:
→ 가로(right - left)는 반드시 줄어듦
→ 세로는 min(h[left], 새 h[right])인데 h[left]보다 커질 수 없음
  (h[left]가 이미 병목)
→ 넓이가 절대로 커질 수 없음

따라서 짧은 쪽(h[left])을 이동해야 더 큰 넓이를 찾을 가능성이 생김
```

---

### 🧩 핵심 접근법 — Two Pointers (O(n))

**접근 순서**

```
Step 1. left = 0, right = height.length - 1
Step 2. left < right인 동안:
    - area = Math.min(height[left], height[right]) * (right - left)
    - maxArea = Math.max(maxArea, area)
    - height[left] < height[right] → left++  (짧은 쪽 이동)
    - 아니면 → right--
Step 3. maxArea 반환
```

**핵심 아이디어 시각화**

```
height = [1, 8, 6, 2, 5, 4, 8, 3, 7]

left=0(h=1), right=8(h=7)
  area = min(1,7) * 8 = 8, maxArea=8
  h[left](1) < h[right](7) → left++

left=1(h=8), right=8(h=7)
  area = min(8,7) * 7 = 49, maxArea=49
  h[left](8) >= h[right](7) → right--

left=1(h=8), right=7(h=3)
  area = min(8,3) * 6 = 18, maxArea=49
  h[left](8) >= h[right](3) → right--

... (계속)

left=1(h=8), right=6(h=8)
  area = min(8,8) * 5 = 40, maxArea=49

최종: 49 ✅
```

---

### 올바른 풀이

```ts
function maxArea(height: number[]): number {
  let left = 0;
  let right = height.length - 1;
  let max = 0;

  while (left < right) {
    const area = Math.min(height[left], height[right]) * (right - left);
    max = Math.max(max, area);

    if (height[left] < height[right]) {
      left++;   // 짧은 쪽 이동
    } else {
      right--;  // 짧은 쪽 이동
    }
  }

  return max;
}
```

| 복잡도 | 값 |
|--------|----|
| 시간 | O(n) — 1회 순회 |
| 공간 | O(1) — 변수 3개만 사용 |

---

### 잘한 점 ✅ / 보완할 점 🔧

**잘한 점**
- "가로 * 세로"라는 넓이 공식을 정확히 파악
- "구간을 어떻게 잡지?"라는 막힌 지점을 명확히 인식 → 이게 바로 Two Pointers의 시작점

**보완할 점**
- 막혔을 때 → "양끝에서 시작하면 어떨까?" 라는 Two Pointers 발상으로 연결
- 짧은 쪽 이동 이유를 설명할 수 있어야 함:
  ```
  긴 쪽을 이동하면 → 가로 감소 + 세로는 짧은 쪽에 의해 제한 → 넓이 불가능하게 커짐
  짧은 쪽을 이동해야 → 더 큰 높이를 만날 가능성이 생김
  ```
- 넓이 공식 한 줄로 외우기:
  ```ts
  Math.min(height[left], height[right]) * (right - left)
  ```
